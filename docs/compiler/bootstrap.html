Zeta 编译器与自举路线

状态：draft

更新时间：2026-05-28

自举原则

自编译是最终目标，但第一版编译器必须使用成熟宿主语言实现。推荐使用 Rust 或 Zig 构建 stage0，以降低早期调试、跨平台、包管理和性能风险。

阶段定义

  • Stage 0：宿主语言实现的 bootstrap 编译器。
  • Stage 1：使用 Zeta 重写 parser 或 typechecker 等前端模块，并接回 stage0。
  • Stage 2：Zeta 编译器可以编译 Zeta 编译器核心前端。
  • Stage 3：完整编译器由 Zeta 编译自身，stage0 仅保留为冷启动 seed。

编译器管线

Zeta Source
  -> Lexer / Parser
  -> AST
  -> Name Resolution / Type Check
  -> HIR
  -> MIR
  -> Backend Interface
  -> LLVM Native / WASM Component / Interpreter

质量闸门

  • Parser golden tests：AST 快照、错误恢复、源码 span。
  • Typecheck golden tests：类型错误、跨模块解析、错误定位。
  • HIR/MIR golden tests：IR dump 稳定，MIR verifier 通过。
  • Backend smoke tests：LLVM/WASM 编译执行结果一致。
  • Bootstrap check：stage1/stage2/stage3 产物行为稳定。
