Zeta Parser / AST / Span 第一版规格

状态：draft

更新时间：2026-05-29

适用范围：Zeta Stage 0 lexer、parser、AST 节点、source span、诊断和 AST golden dump。

验收标准：worker 可以据此实现或审查 Stage 0 parser 原型；语法范围与 MVP baseline 一致；非 MVP 语法必须标记 deferred。

范围边界

本规格冻结 Stage 0 第一版 parser 表面，不定义完整语言。任何新增语法必须先更新本文件、compiler corpus 和对应 golden dump，再进入实现。

  • Included：module/import/export、struct、enum、fn、参数、返回类型、let/let mut、赋值、return、if/else、while、match、函数调用、基础字面量和整数算术。
  • Deferred：宏、泛型参数、trait/impl、闭包、async/await、for、完整 pattern matching、切片视图、错误恢复型 AST 节点。

Token 与 Span

项目 规则
Span 单位 Span.start 与 Span.end 使用源文件 UTF-8 byte offset，半开区间 [start, end)。
Token span 每个 token 保留原始源码范围；字符串 token 的 span 覆盖引号；EOF span 位于源码末尾。
Diagnostic span parser 诊断指向当前 token；lexer 诊断指向非法 token；resolver/typecheck 对已有 AST span 的名字、赋值目标和表达式报错必须指向真实源码范围；尚未携带 span 的聚合错误才允许临时使用 0..0。
路径 span module/import path 在 AST 中先保存规范化字符串或字符串数组；完整 path span 后续随 symbol table 引入。

AST 节点

层级 节点 字段
Module Module items: Vec<Item>
Item ModuleDecl / Import name: String；path: Vec<String>
Item Struct / Enum exported、name、fields 或 variants
Item Function exported、name、params、return_type、body
Stmt Let / Assign mutable、name、ty、value；赋值只允许 identifier target。
Stmt If / While / Match condition、then_body、else_body、body、arms
Stmt Return / Expr Return(None) 表示无值 return；表达式语句必须以分号结束。
Expr Name、Int、String、Bool、Binary、Unary、Call 二元操作支持算术、比较、&& 和 ||；一元操作支持 !；call callee 当前只允许函数名。
Pattern Name、Int、String、Bool、Wildcard Stage 0 match arm pattern 不做解构；复杂 pattern deferred。

表达式优先级

  1. primary：identifier、literal、parenthesized expression。
  2. call：callee(args...)，callee 必须是 Name。
  3. multiplicative：*、/，左结合。
  4. additive：+、-，左结合。

错误恢复策略

  • Stage 0 parser 默认 fail-fast：遇到 parser diagnostic 后返回单个错误，不构造部分 AST。
  • REPL 输入先包装成临时函数；空行、顶层 item 和表达式行使用不同包装策略。
  • 后续错误恢复必须以 corpus 证明：同文件后续顶层 item 可以继续解析，且不会吞掉有效诊断。
  • 错误恢复节点、missing token 节点和 trivia 保留不进入当前 AST。

AST Golden Dump

AST dump 是 Stage 0 parser 的稳定外部契约。字段顺序、缩进和节点名变更必须进入 review。

Module
  Function name=main exported=false
    Let name=value type=Int mutable=true
      Int 1
    Assign name=value
      Binary op=add
        Name value
        Int 1
  • 缩进固定为两个空格。
  • 每个节点独占一行；子节点按源码顺序输出。
  • 布尔字段显式输出 true / false；mutable=false 可省略。
  • 字符串使用 debug 转义格式，避免换行和引号破坏 golden。
  • dump 禁止包含内存地址、hash map 遍历顺序、本地路径和当前时间。

质量闸门

  • cargo test 必须覆盖核心 AST dump、错误诊断、REPL 包装和新语法节点。
  • python3 tools/check-docs.py 必须验证本文链接和元信息。
  • 新增 AST 节点必须同步更新 parser、dump、resolver/typecheck 处理策略和文档。
  • 任何 deferred 语法只能出现在设计讨论中，不得进入 Stage 0 parser 接受路径。
