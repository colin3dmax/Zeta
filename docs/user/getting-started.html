Zeta 用户快速开始

状态：draft

更新时间：2026-05-28

适用范围：Zeta Stage 0 CLI、AST dump、语义检查、REPL 原型和本地示例文件。

验收标准：用户可以运行示例、检查源码、进入 REPL，并知道当前 REPL 只解析和检查语法，不执行程序。

安装前提

  • 需要 Rust 和 Cargo。
  • 当前 Stage 0 编译器从源码运行，不提供预编译安装包。
  • 命令默认从仓库根目录执行。

验证环境

cargo test
python3 tools/check-docs.py
python3 tools/check-vscode-extension.py

查看 AST

ast-dump 会解析 Zeta 源码并输出稳定 AST dump，适合调试 parser 和 golden corpus。

cargo run -- ast-dump testdata/core_items.zeta

检查源码

check 当前执行 parser、最小 name resolution 和基础 typecheck。

cargo run -- check testdata/core_items.zeta

当前检查范围包括重复定义、未知名字、Int / String / Bool、算术表达式、let 注解、if / while 条件和 return 类型。

进入 REPL

cargo run -- repl

当前 REPL 是 Stage 0 交互式语法终端：它实时解析输入，并输出 AST dump。真正执行 Zeta 代码需要后续 MIR interpreter。

REPL 命令

命令 用途 示例
:help 显示 REPL 命令和主题列表。 :help
:doc <topic> 查询关键词或类型的短文档。 :doc let
:complete <prefix> 列出当前已知补全候选。 :complete st
:quit 退出 REPL。 :quit

示例输入

let answer: Int = 40 + 2;
:doc struct
:complete Str
:quit

当前限制

  • REPL 当前不执行 Zeta 程序，只输出 AST dump。
  • 类型检查只覆盖 Stage 0 的最小标量和控制流。
  • 错误 span 仍是早期实现，后续会补充源码位置。
  • 真正的实时运行需要 HIR/MIR lowering 和 MIR interpreter。
