Zeta 用户快速开始

状态：draft

更新时间：2026-05-29

适用范围：Zeta Stage 0 CLI、AST/HIR/MIR dump、语义检查、MIR interpreter、REPL 原型和本地示例文件。

验收标准：用户可以运行示例、检查源码、执行无参数 main，并在 REPL 里直接计算表达式。

安装前提

  • 需要 Rust 和 Cargo。
  • 当前 Stage 0 编译器从源码运行，不提供预编译安装包。
  • 命令默认从仓库根目录执行。

验证环境

cargo test
python3 tools/check-docs.py
python3 tools/check-vscode-extension.py

查看 AST

ast-dump 会解析 Zeta 源码并输出稳定 AST dump，适合调试 parser 和 golden corpus。

cargo run -- ast-dump testdata/core_items.zeta

查看 HIR

hir-dump 会先通过 name resolution 和 typecheck，再输出稳定 HIR 文本，作为后续 MIR 和 backend golden tests 的输入基线。

cargo run -- hir-dump testdata/core_items.zeta

查看 MIR

mir-dump 当前覆盖 Stage 0 可运行子集，会输出 locals、temps、store、return、call 和基础控制流的稳定 MIR 文本。

cargo run -- mir-dump testdata/run_mut.zeta

检查源码

check 当前执行 parser、最小 name resolution 和基础 typecheck。

cargo run -- check testdata/core_items.zeta

当前检查范围包括重复定义、未知名字、Int / String / Bool、算术表达式、比较表达式、布尔逻辑表达式、函数调用、let 注解、let mut 可变绑定、赋值语句、if / while 条件和 return 类型。

执行程序

run 会先 lower 到结构化 MIR，再通过 Stage 0 MIR interpreter 执行无参数 main；当前支持整数算术、比较运算、布尔逻辑、函数调用、let、let mut、赋值、return 和基础 if / while。

cargo run -- run testdata/run_basic.zeta
cargo run -- run testdata/run_call.zeta
cargo run -- run testdata/run_compare.zeta
cargo run -- run testdata/run_bool_logic.zeta
cargo run -- run testdata/run_struct.zeta
cargo run -- run testdata/run_enum.zeta

可变绑定和赋值

Zeta 默认局部绑定不可变。需要在后续语句里重新赋值时，声明时必须写 let mut；否则 check 会报告 RESOLVE_ASSIGN_IMMUTABLE。

fn main() -> Int {
  let mut answer: Int = 40;
  answer = answer + 2;
  return answer;
}
cargo run -- run testdata/run_mut.zeta

比较表达式

比较运算返回 Bool，可直接作为 if 和 while 条件。<、<=、>、>= 当前只支持 Int；== 和 != 要求左右类型一致。

fn main() -> Int {
  let mut count: Int = 0;
  while count < 3 {
    count = count + 1;
  }
  if count == 3 {
    return 42;
  }
  return 0;
}

布尔逻辑

&&、|| 和 ! 用于组合或取反 Bool 表达式；&& 和 || 使用短路语义。

fn main() -> Int {
  let ready: Bool = true;
  let done: Bool = false;
  if ready && !done {
    return 42;
  }
  return 0;
}

标量 match

match 当前支持 Int、String、Bool 字面量模式和 _ 通配模式，分支体可以直接 return。

fn main() -> Int {
  let value: Int = 2;
  match value {
    1 -> { return 10; },
    2 -> { return 42; },
    _ -> { return 0; },
  }
  return 0;
}
cargo run -- run testdata/run_match.zeta

结构体值和字段访问

struct 可以声明记录类型；运行时当前支持结构体字面量和 value.field 字段访问。结构体字面量必须提供声明中的全部字段，字段值类型必须匹配声明。

struct User {
  name: String,
  age: Int,
}

fn main() -> Int {
  let user: User = User { name: "Ada", age: 42 };
  return user.age;
}
cargo run -- run testdata/run_struct.zeta

枚举变体和 match

enum 当前支持限定变体值，例如 ResultTag.Ok。枚举值可以赋给局部变量，也可以在 match 中用同样的限定变体模式分支。

enum ResultTag {
  Ok,
  Err,
}

fn main() -> Int {
  let tag: ResultTag = ResultTag.Ok;
  match tag {
    ResultTag.Ok -> { return 42; },
    ResultTag.Err -> { return 0; },
  }
  return 0;
}
cargo run -- run testdata/run_enum.zeta

在线 Playground

官网的 Playground 直接加载 Zeta 编译器前端编译出的 zeta.wasm，不是 JavaScript 模拟器。当前支持三个模式：

按钮 能力 适用场景
AST 解析源码并输出 AST dump。 确认语法结构和 parser 行为。
Check 执行 parser、name resolution 和基础 typecheck。 检查重复定义、未知名字、类型不匹配和 return 类型。
Run 执行无参数 main 并返回 Stage 0 interpreter 结果。 在线试运行整数算术、函数调用和基础控制流。 
module demo.core;

export fn main() -> Int {
  let mut answer: Int = 40;
  answer = answer + 2;
  return answer;
}

进入 REPL

cargo run -- repl

当前 REPL 是 Stage 0 交互式语法终端：表达式会直接执行并返回结果，语句会执行并输出 ok。真实 TTY 下会启用无依赖 line editor，支持输入时语法高亮、Tab 补全、hint、历史上下切换和左右光标移动。

REPL 默认会根据语言偏好显示中文或英文：优先读取 ZETA_LANG 环境变量，其次读取 ~/.zeta/config.toml 里的 language 或 lang，最后根据系统 LANG / LC_ALL 判断。简体中文环境默认显示中文。会话内可以用 :lang zh 或 :lang en 临时切换。

export ZETA_LANG=zh
cargo run -- repl
mkdir -p ~/.zeta
printf 'language = "zh"\n' > ~/.zeta/config.toml

高级 REPL

高级 REPL 使用可选 reedline 依赖，默认不启用，避免影响离线和跨平台基础构建。

cargo run --features repl-rich -- repl

如果当前终端不支持 reedline 需要的 cursor 查询，高级 REPL 会自动退回内置 line editor，继续保留基础高亮、补全和历史能力。

需要通过代理安装依赖时，可以先设置代理环境变量：

export https_proxy=http://127.0.0.1:33210
export http_proxy=http://127.0.0.1:33210
export all_proxy=socks5://127.0.0.1:33211
cargo test --features repl-rich

REPL 命令

命令 用途 示例
:help 显示 REPL 命令和主题列表。 :help
:doc <topic> 查询关键词或类型的短文档。 :doc let
:complete <prefix> 列出当前已知补全候选。 :complete st
:api 查看当前 Stage 0 API 和标准库命名空间概览。 :api
:topics 列出终端内置文档主题。 :topics
:examples 显示可直接运行的示例。 :examples
:lang zh|en 切换当前 REPL 会话语言。 :lang zh
:quit 退出 REPL。 :quit

示例输入

40 + 2
let answer: Int = 40 + 2;
let mut count: Int = 1;
count = count + 1;
true && !false;
:doc struct
:complete Str
:quit

当前限制

  • 默认 REPL 使用无依赖 line editor；启用 repl-rich feature 后会使用 reedline。
  • 管道输入和非 TTY 环境会自动退回普通行输入。
  • 类型检查只覆盖 Stage 0 的最小标量、函数调用和控制流。
  • 诊断已经输出行列和源码片段；跨文件诊断和更复杂恢复策略仍会继续补齐。
  • 当前 run 已使用 Stage 0 MIR interpreter；REPL 会话仍保留 AST session，以维持跨行局部变量状态。
