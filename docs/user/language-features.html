模块、import 和 export

module 声明当前文件模块名；export 标记函数或类型对外可见。单文件检查时，import 只接受标准 API 边界；目录检查或运行时，CLI 会递归读取目录内的 .zeta 文件，建立最小 module graph，并允许导入同批检查中的本地模块。

当前 resolver 接受 std.core、std.io 和 module graph 中存在的本地模块。导入本地模块后，可以直接调用该模块的 export fn，也可以使用 demo.math.answer() 这类限定调用写清来源；如果模块路径很长，可以写 import demo.math as math;，然后调用 math.answer()。未导出的函数不会进入当前文件作用域。未知路径会报告 RESOLVE_UNKNOWN_IMPORT，重复模块会报告 RESOLVE_DUPLICATE_MODULE。如果多个导入模块导出同名函数，短名调用会报告 RESOLVE_AMBIGUOUS_FUNCTION，需要改用限定调用或 alias 调用。目录级 run 要求同批文件里只有一个 main，并会执行跨文件导出的函数调用。

module demo.core;
import std.core;
import std.io;

export fn main() -> Int {
  return 42;
}
import std.net; // RESOLVE_UNKNOWN_IMPORT
// testdata/modules_ok/main.zeta
module demo.app;
import demo.math;
import std.core;

fn main() -> Int {
  return answer();
}
// testdata/modules_ok/math.zeta
module demo.math;

export fn answer() -> Int {
  return 42;
}
cargo run -- check testdata/modules_ok
cargo run -- run testdata/modules_ok

限定调用适合跨模块函数名可能重复的工程。下面的 main 明确调用 demo.math 模块导出的 answer，而 answer 内部仍可以调用同模块私有 helper。

// testdata/modules_qualified/main.zeta
module demo.app;
import demo.math;

fn main() -> Int {
  return demo.math.answer();
}
// testdata/modules_qualified/math.zeta
module demo.math;

export fn answer() -> Int {
  return helper();
}

fn helper() -> Int {
  return 42;
}
cargo run -- run testdata/modules_qualified

别名调用适合保留来源信息，同时避免在调用点重复完整模块路径。

// testdata/modules_alias/main.zeta
module demo.app;
import demo.math as math;

fn main() -> Int {
  return math.answer();
}
// testdata/modules_alias/math.zeta
module demo.math;

export fn answer() -> Int {
  return 42;
}
cargo run -- run testdata/modules_alias
import demo.math;
import demo.more;

fn main() -> Int {
  return answer(); // RESOLVE_AMBIGUOUS_FUNCTION
}

标量与表达式

当前标量类型包括 Int、Float、String 和 Bool（Float 详见下文）。整数算术支持 + - * /，乘除优先级高于加减。

fn main() -> Int {
  return 1 + 2 * 20 + 1;
}

比较表达式返回 Bool。<、<=、>、>= 当前只接受 Int；== 和 != 要求左右类型一致。

fn main() -> Bool {
  return 40 + 2 == 42;
}

布尔逻辑支持 &&、|| 和 !，其中 && / || 使用短路语义。

fn main() -> Bool {
  return true && !false;
}

let、let mut 和赋值

Zeta 默认局部绑定不可变。需要后续赋值时，必须在声明处写 let mut。给不可变绑定赋值会被 resolver/typecheck 拒绝。

fn main() -> Int {
  let mut answer: Int = 40;
  answer = answer + 2;
  return answer;
}

函数、参数和 return

fn 声明函数，参数和返回值使用显式类型。run 当前执行无参数 main。函数调用会检查参数数量和参数类型。

fn add(left: Int, right: Int) -> Int {
  return left + right;
}

fn main() -> Int {
  return add(40, 2);
}

if/else、while、break 和 continue

if 和 while 的条件必须是 Bool。比较表达式和布尔逻辑可以直接作为条件。break 和 continue 只能出现在 while 循环体内。

fn main() -> Int {
  let mut count: Int = 0;
  while count < 3 {
    count = count + 1;
    if count == 2 {
      continue;
    }
  }
  if count == 3 && !false {
    return 42;
  }
  return 0;
}

数组字面量、下标和 len

Stage 0 当前支持同质数组类型 IntArray、StringArray 和 BoolArray。数组字面量必须非空，元素类型必须一致；下标表达式使用 Int，.len 返回数组长度。

fn main() -> Int {
  let values: IntArray = [2, 4, 6];
  return values[0] + values[1] + values.len;
}

当前数组能力用于支撑自举编译器的 token/span 列表和小型表结构；还没有泛型 Vec、数组拼接、切片视图或用户自定义元素类型数组。

std.core typed array builder

导入 std.core 后，可以用 typed builder 函数创建空数组并追加元素：int_array_empty / int_array_push、string_array_empty / string_array_push、bool_array_empty / bool_array_push。这些函数返回新数组；需要多次追加时配合 let mut 和赋值使用。

import std.core;

fn main() -> Int {
  let mut values: IntArray = int_array_empty();
  values = int_array_push(values, 2);
  values = int_array_push(values, 4);
  values = int_array_push(values, 6);
  return values[0] + values[1] + values.len;
}

这不是完整泛型 Vec，而是 Stage 0/1 自举所需的最小容器折中。

std.core 字符串 byte 扫描

导入 std.core 后，可以使用自举 lexer/parser 需要的最小字符串扫描函数：string_len 返回 UTF-8 byte 长度，string_byte_at 用 Int 下标读取 byte 并以 Int 返回，string_byte_slice 使用 byte 起点和 byte 长度截取字符串，ASCII 分类函数接收 byte Int 并返回 Bool。Stage 1 Zeta 前端种子已用这些 API 对齐 Rust Stage 0 的 // 行注释跳过行为。

import std.core;

fn main() -> Int {
  let text: String = "A9 zeta";
  let first: Int = string_byte_at(text, 0);
  let digit: Int = string_byte_at(text, 1);
  if ascii_is_alpha(first) && ascii_is_digit(digit) {
    return string_len(text) + digit;
  }
  return 0;
}

当前刻意不引入 Char、Byte 或 Unicode iterator；完整 Unicode 文本处理会在更成熟的标准库阶段单独设计。

std.core 字符串构造

导入 std.core 后，可以使用 string_concat 拼接两个字符串，并用 int_to_string 把整数格式化为字符串。这是完整格式化库之前的最小能力，用于 Stage1/Stage2 前端生成稳定文本摘要和 dump。

import std.core;

fn main() -> String {
  return string_concat("score=", int_to_string(42));
}

std.io 文件、路径和诊断

导入 std.io 后，可以使用自举编译器需要的最小 IO 基础：file_read_to_string 读取源码文件并返回 ResultString.Ok(text) 或 ResultString.Err(message)；path_join 和 path_basename 处理简单路径片段；diagnostic_format 生成稳定诊断字符串。wasm 目标不会访问宿主文件系统，文件读取会返回 Err。

import std.io;

fn main() -> String {
  let path: String = path_join("src", "main.zeta");
  return diagnostic_format("LEX_BAD_CHAR", 3, 5, path_basename(path));
}

struct、enum 和 match

struct 当前支持结构体字面量和字段访问，适合先表达稳定记录数据。enum 当前支持限定变体值，例如 ResultTag.Ok。match 可以对 Int、String、Bool、枚举变体和 _ 通配模式执行分支。

struct User {
  name: String,
  age: Int,
}

fn main() -> Int {
  let user: User = User { name: "Ada", age: 42 };
  return user.age;
}
fn main() -> Int {
  let value: Int = 2;
  match value {
    1 -> { return 10; },
    2 -> { return 42; },
    _ -> { return 0; },
  }
  return 0;
}
module demo.data;

export struct User {
  name: String,
  age: Int,
}

enum ResultTag {
  Ok,
  Err,
}
enum ResultTag {
  Ok,
  Err,
}

fn main() -> Int {
  let tag: ResultTag = ResultTag.Ok;
  match tag {
    ResultTag.Ok -> { return 42; },
    ResultTag.Err -> { return 0; },
  }
  return 0;
}

Float 浮点

Float 是 64 位 IEEE-754 浮点标量，字面量写作 1.5、2.0（小数点后必须有数字，以区分字段访问 x.0 和区间 1..2）。算术 + - * / 支持 Int 或 Float，但两者不隐式混用；取模 % 和位运算仅限 Int。比较返回 Bool。打印时整数值也带小数点（4.0 而非 4）。

fn scale(x: Float, k: Float) -> Float {
  return x * k;
}

fn main() -> Int {
  let r: Float = scale(1.5, 4.0);
  if r > 5.9 { if r < 6.1 { return 1; } }
  return 0;
}

Float 在原生后端编译为 LLVM double（fadd/fsub/fmul/fdiv/fcmp），与解释器逐位对齐。

Tuple 元组

元组是固定元数的异构聚合：字面量 (a, b)（带逗号；(a) 仍是普通分组），按位置索引 t.0、t.1，可嵌套 t.1.0。类型推断在函数内即可用；也支持类型注解 (Int, String)，因此元组能跨函数边界传参与返回。

fn pair(a: Int, b: Int) -> (Int, Int) {
  return (a, b);
}

fn main() -> Int {
  let t = pair(10, 20);
  return t.0 + t.1;
}

原生后端把元组实现为 LLVM 匿名结构体 { T0, T1 }（insertvalue/extractvalue），按值语义传递。

Closure 闭包

Lambda 写作 |x: Int| body（表达式体，参数带类型；|| body 为零参）。闭包是一等值，按值捕获创建时刻的自由变量（快照语义）。函数类型写作 fn(Int) -> Int，可作参数/返回类型，支持高阶函数与返回闭包。

fn adder(n: Int) -> fn(Int) -> Int {
  return |x: Int| x + n;
}

fn main() -> Int {
  let add7 = adder(7);
  return add7(35);
}

原生后端用闭包转换实现：lambda 提升为带堆分配环境（env）参数的顶层函数，闭包值为 { 函数指针, env 指针 }，间接调用解包后跳转。

Generics 泛型

泛型函数写作 fn id<T>(x: T) -> T，类型参数从实参推断（调用点无需 <>）。类型参数在函数体内是不透明类型——只能做与类型无关的操作（传递、存入元组/数组、返回），对 T 做算术会被类型检查拒绝。

fn pick<A, B>(a: A, b: B) -> A {
  return a;
}

fn main() -> Int {
  return pick(9, true);
}

原生后端通过单态化实现：每个具体类型实例（如 id$Int、id$Float）按需生成一份专门副本并缓存；解释器因运行时动态类型，泛型函数无需单态化即可直接运行。当前仅支持泛型函数，尚不支持泛型 struct/enum。

CLI、REPL 和编辑器

CLI 提供 ast-dump、hir-dump、mir-dump、check、run 和 repl。REPL 支持表达式执行、语句执行、文档查询、补全和输入着色。
