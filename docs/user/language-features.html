模块、import 和 export

module 声明当前文件模块名；import 先作为模块路径语法进入 AST；export 标记函数或类型对外可见。当前 Stage 0 还没有跨文件模块图。

module demo.core;
import std.io;

export fn main() -> Int {
  return 42;
}

在 Playground 测试模块示例

标量与表达式

当前标量类型包括 Int、String 和 Bool。整数算术支持 + - * /，乘除优先级高于加减。

fn main() -> Int {
  return 1 + 2 * 20 + 1;
}

比较表达式返回 Bool。<、<=、>、>= 当前只接受 Int；== 和 != 要求左右类型一致。

fn main() -> Bool {
  return 40 + 2 == 42;
}

布尔逻辑支持 &&、|| 和 !，其中 && / || 使用短路语义。

fn main() -> Bool {
  return true && !false;
}

let、let mut 和赋值

Zeta 默认局部绑定不可变。需要后续赋值时，必须在声明处写 let mut。给不可变绑定赋值会被 resolver/typecheck 拒绝。

fn main() -> Int {
  let mut answer: Int = 40;
  answer = answer + 2;
  return answer;
}

在 Playground 测试绑定和赋值

函数、参数和 return

fn 声明函数，参数和返回值使用显式类型。run 当前执行无参数 main。函数调用会检查参数数量和参数类型。

fn add(left: Int, right: Int) -> Int {
  return left + right;
}

fn main() -> Int {
  return add(40, 2);
}

在 Playground 测试函数调用

if/else 和 while

if 和 while 的条件必须是 Bool。比较表达式和布尔逻辑可以直接作为条件。

fn main() -> Int {
  let mut count: Int = 0;
  while count < 3 {
    count = count + 1;
  }
  if count == 3 && !false {
    return 42;
  }
  return 0;
}

struct、enum 和 match

struct 和 enum 当前支持 parse、AST dump 和 HIR dump，用于冻结数据建模语法。结构体值、字段访问、枚举变体值和 match 运行时语义会在后续阶段接入。

module demo.data;

export struct User {
  name: String,
  age: Int,
}

enum ResultTag {
  Ok,
  Err,
}

在 Playground 查看数据声明 AST

CLI、REPL 和编辑器

CLI 提供 ast-dump、hir-dump、mir-dump、check、run 和 repl。REPL 支持表达式执行、语句执行、文档查询、补全和输入着色。

zeta ast-dump testdata/core_items.zeta
zeta check testdata/core_items.zeta
zeta run testdata/run_bool_logic.zeta
zeta repl
