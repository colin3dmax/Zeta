Zeta 本地安装

状态：accepted

更新时间：2026-05-30

适用范围：Zeta Stage 0 命令行工具、REPL、官网安装入口和当前平台本地打包方法。

验收标准：用户可以在 macOS、Linux 或 Windows 上从源码构建 Zeta，运行 zeta repl、zeta check 和 zeta run。

Install

当前推荐从源码安装；release 打包脚本已经可复用。

Zeta 还处在 Stage 0，命令行工具已经可本地运行。官网提供源码安装、本地 release 构建和脚本化打包方法；预构建下载会在发布机或 CI 跑同一套脚本后开放。

查看已发布 CLI 包

平台状态

平台 当前可用方式 预构建安装包
macOS arm64 / x86_64 源码构建、本地 release 构建、tools/package-release.sh 打包。 脚本已支持；稳定下载待 release 发布。
Linux x86_64 / aarch64 源码构建、本地 release 构建、指定 Rust target 打包。 脚本已支持；需要本机安装对应 Rust target/linker。
Windows x86_64 通过 Rust/Cargo 在 PowerShell 或 Git Bash 中源码构建；发布机可指定 Windows target。 脚本会优先生成 .zip；需要 Windows target 和 linker。

准备 Rust

安装 Rust 后确认 cargo 可用。

rustc --version
cargo --version

从源码运行

git clone https://github.com/colin3dmax/Zeta.git
cd Zeta
cargo run -- repl

构建本地 release

cargo build --release
./target/release/zeta repl
./target/release/zeta check testdata/core_items.zeta
./target/release/zeta run testdata/run_bool_logic.zeta

安装到 PATH

macOS / Linux 可以把 release 二进制复制到用户本地 bin 目录。

mkdir -p ~/.local/bin
cp target/release/zeta ~/.local/bin/zeta
export PATH="$HOME/.local/bin:$PATH"
zeta repl

Windows PowerShell 可以直接运行 release 产物，或把 target\release 加入用户 Path。

cargo build --release
.\target\release\zeta.exe repl

生成安装包

打包脚本会构建 release，并把 zeta、README、HTML 文档、INSTALL.txt 和校验信息打进 dist/packages/。

sh tools/package-release.sh
ls dist/packages

发布机可以指定已经安装好的 Rust target。脚本不会自动下载 target 或 linker，避免在用户机器上隐式改变工具链。

rustup target add x86_64-unknown-linux-gnu
sh tools/package-release.sh --target x86_64-unknown-linux-gnu

rustup target add aarch64-apple-darwin
sh tools/package-release.sh --target aarch64-apple-darwin

rustup target add x86_64-pc-windows-msvc
sh tools/package-release.sh --target x86_64-pc-windows-msvc

兼容旧入口：

sh tools/package-local.sh

验证安装

zeta ast-dump testdata/core_items.zeta
zeta check testdata/core_items.zeta
zeta run testdata/run_bool_logic.zeta
zeta repl

VS Code 插件

编辑器插件当前通过本地脚本安装。

sh editors/vscode-zeta/scripts/install-local.sh

更多说明见 VS Code 插件使用说明。
