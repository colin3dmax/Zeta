Zeta AI 原生能力与权限模型

状态：draft

更新时间：2026-05-28

核心边界

  • Capability：描述代码被授权做什么，例如文件读取、网络访问、模型调用。
  • Schema：描述输入、输出、错误和可验证数据契约。
  • Tool：描述可调用能力的函数边界、权限、超时、幂等性和失败场景。
  • Agent：描述有状态任务单元的决策边界、工具集合和策略约束。

设计原则

  • 默认无权限：代码默认不能访问文件、网络、系统命令、环境变量、模型服务或外部工具。
  • 声明先于调用：所有 capability 必须在函数、tool、agent 或模块边界显式声明。
  • Schema 先行：AI 输出、tool 输入输出和 agent 状态必须先有可验证 schema，再允许进入实现。
  • 最小授权：授权粒度应缩到具体路径、域名、模型族、工具名和操作类型。
  • 可审计：每次 capability 调用必须能生成 trace 事件，记录调用方、权限、输入摘要、输出摘要、错误和耗时。
  • 可降级：权限拒绝、超时、模型不可用和 schema 校验失败必须进入显式错误模型。

Capability 生命周期

  1. 声明：源码通过 uses 或模块 manifest 声明需要的 capability。
  2. 发现：编译器和包管理器汇总 capability 清单，生成可审计权限图。
  3. 授权：运行时或部署配置决定具体授权范围，例如路径、域名、模型和调用额度。
  4. 调用：代码只能通过已授权 capability 入口访问外部能力。
  5. 审计：运行时记录 trace，供测试、生产观测和安全审查使用。
  6. 失效：权限可以因配置变更、过期、额度耗尽、策略拒绝或用户撤销而失效。

Capability 命名

Capability 使用点分层级命名，一级为能力域，二级为操作，后续层级用于细分 provider 或资源类型。

fs.read
fs.write
net.connect
net.listen
env.read
process.spawn
ai.chat
ai.embedding
ai.vision
tool.call
secret.read

权限声明

fn summarize(path: Path) -> Result<String, Error>
  uses [fs.read, ai.chat]
{
  let text = fs.readText(path)?
  ai.chat(model: "fast", input: text)
}

模块级 manifest 可以收窄授权范围。源码声明“需要什么”，部署配置声明“允许到什么范围”。

[permissions]
fs.read = ["./docs", "./data/public"]
net.connect = ["api.example.com:443"]
ai.chat = {
  models = ["fast", "reasoning"],
  max_tokens_per_call = 4096
}

Schema 契约字段

字段 要求 说明
name 必填 稳定名称，使用 PascalCase。
version 必填 语义版本。破坏性变更必须提升 major。
fields 必填 字段名、类型、可选性、默认值和验证规则。
examples 必填 至少一个成功样例和一个失败样例。
compatibility 必填 向前兼容规则、废弃字段和迁移策略。

Tool 契约字段

字段 要求 说明
input 必填 输入 schema，禁止未结构化 Any 作为正式接口。
output 必填 输出 schema，AI 输出必须通过校验。
errors 必填 错误枚举，必须区分权限、超时、校验和外部服务失败。
uses 必填 最小 capability 集合。
timeout 必填 默认超时和最大超时。
idempotency 必填 标记为 none、read、retry_safe 或 key_required。
trace 必填 调用链字段、脱敏策略和采样策略。

Agent 契约字段

字段 要求 说明
tools 必填 Agent 可调用 tool 白名单。
memory 必填 状态存储类型、生命周期和隐私策略。
policy 必填 决策边界、人类确认点和禁止行为。
budget 必填 最大调用次数、token、费用、时间和重试次数。
explainability 必填 需要记录的计划、选择理由和最终决策摘要。

示例

schema Task {
  title: String
  priority: "low" | "medium" | "high"
}

tool classify(input: String) -> Result<Task, AIError>
  uses [ai.chat]
{
  ai.extract<Task>(model: "fast", input: input)
}

Agent 示例

agent SupportAgent {
  memory: SessionMemory(ttl: "24h")
  tools: [searchDocs, createTicket]
  policy: [
    "no_payment_action_without_user_confirm",
    "no_secret_exfiltration",
    "human_review_required_for_account_closure"
  ]
  budget: {
    max_tool_calls: 12,
    max_duration_ms: 30000
  }

  async fn handle(msg: Message) -> Result<Response, AgentError>
    uses [tool.call, ai.chat]
  {
    plan(msg).execute()
  }
}

错误模型

错误 含义 是否可恢复
PermissionDenied 未声明或未授权 capability。 可通过配置或用户授权恢复。
SchemaInvalid 输入或输出不符合 schema。 通常可通过重试、修复输入或降级恢复。
Timeout 超过 tool 或 agent 的时间限制。 可重试，但必须受预算限制。
ProviderUnavailable 外部模型、网络或工具服务不可用。 可降级到备用 provider 或排队。
PolicyRejected 调用违反 agent policy 或部署策略。 需要用户确认、策略变更或终止任务。
BudgetExceeded 超过费用、token、调用次数或时间预算。 需要新预算或人工确认。

版本策略

  • Schema、tool 和 agent 契约使用语义版本。
  • 新增可选字段是 minor 变更。
  • 删除字段、改变字段含义、收紧验证规则、改变错误语义是 major 变更。
  • 破坏性变更必须提供迁移说明、兼容窗口和回滚方案。

Lint 与 Review 规则

  • 禁止 tool 缺少 schema、错误模型、timeout 或 uses 声明。
  • 禁止 agent 调用未列入 tools 白名单的 tool。
  • 禁止 capability 使用通配符授权进入 stable 文档。
  • 禁止 AI 输出未经 schema 校验进入业务状态。
  • Review 必须检查成功样例、失败样例、权限最小化和 trace 脱敏。

质量要求

  • Tool 必须声明 schema、错误模型、权限和超时。
  • Agent 必须声明可调用 tool 和决策边界。
  • 权限默认最小化，禁止隐式文件、网络、系统命令和 AI 模型访问。
  • Review 必须包含成功样例和失败样例。
  • Done 必须附带测试结果、文档链接和 DevGame 验收证据。
