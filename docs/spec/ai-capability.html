Zeta AI 原生能力与权限模型

状态：draft

更新时间：2026-05-28

适用范围：Zeta AI 原生 capability、schema、tool、agent、policy、budget、授权配置和 trace 审计契约。

验收标准：能力调用必须有声明、授权、资源收窄、错误码和脱敏 trace；stable 文档禁止无边界通配符。

MVP 边界：capability、schema、tool、policy、budget 和 trace 的冻结范围以 Zeta MVP Baseline 与冻结边界 为准。

核心边界

  • Capability：描述代码被授权做什么，例如文件读取、网络访问、模型调用。
  • Schema：描述输入、输出、错误和可验证数据契约。
  • Tool：描述可调用能力的函数边界、权限、超时、幂等性和失败场景。
  • Agent：描述有状态任务单元的决策边界、工具集合和策略约束。

设计原则

  • 不进入核心 runtime：AI provider SDK、agent 框架、模型适配器和高级编排通过标准库模块或第三方包引入，不能扩大语言内核。
  • 默认无权限：代码默认不能访问文件、网络、系统命令、环境变量、模型服务或外部工具。
  • 声明先于调用：所有 capability 必须在函数、tool、agent 或模块边界显式声明。
  • Schema 先行：AI 输出、tool 输入输出和 agent 状态必须先有可验证 schema，再允许进入实现。
  • 最小授权：授权粒度应缩到具体路径、域名、模型族、工具名和操作类型。
  • 可审计：每次 capability 调用必须能生成 trace 事件，记录调用方、权限、输入摘要、输出摘要、错误和耗时。
  • 可降级：权限拒绝、超时、模型不可用和 schema 校验失败必须进入显式错误模型。

Capability 生命周期

  1. 声明：源码通过 uses 或模块 manifest 声明需要的 capability。
  2. 发现：编译器和包管理器汇总 capability 清单，生成可审计权限图。
  3. 授权：运行时或部署配置决定具体授权范围，例如路径、域名、模型和调用额度。
  4. 调用：代码只能通过已授权 capability 入口访问外部能力。
  5. 审计：运行时记录 trace，供测试、生产观测和安全审查使用。
  6. 失效：权限可以因配置变更、过期、额度耗尽、策略拒绝或用户撤销而失效。

Capability 命名

Capability 使用点分层级命名，一级为能力域，二级为操作，后续层级用于细分 provider 或资源类型。

fs.read
fs.write
net.connect
net.listen
env.read
process.spawn
ai.chat
ai.embedding
ai.vision
tool.call
secret.read

权限声明

fn summarize(path: Path) -> Result<String, Error>
  uses [fs.read, ai.chat]
{
  let text = fs.readText(path)?
  ai.chat(model: "fast", input: text)
}

模块级 manifest 可以收窄授权范围。源码声明“需要什么”，部署配置声明“允许到什么范围”。

[permissions]
fs.read = ["./docs", "./data/public"]
net.connect = ["api.example.com:443"]
ai.chat = {
  models = ["fast", "reasoning"],
  max_tokens_per_call = 4096
}

Capability Manifest 示例

Manifest 是包、模块或 agent 的静态声明文件，只描述“需要什么能力”和“默认收窄边界”，不直接授予生产权限。

{
  "schema": "zeta.capability.manifest.v1",
  "name": "docs.summarizer",
  "version": "0.3.0",
  "owner": "ai-runtime",
  "entry": "src/docs/summarizer.zeta",
  "capabilities": [
    {
      "id": "fs.read",
      "reason": "读取用户选择的文档片段",
      "resources": {
        "paths": ["./docs/**/*.md", "./data/public/**/*.txt"]
      }
    },
    {
      "id": "ai.chat",
      "reason": "生成结构化摘要",
      "resources": {
        "models": ["fast"],
        "max_tokens_per_call": 4096
      }
    }
  ],
  "tools": ["summarizeDoc"],
  "trace": {
    "redaction_profile": "standard",
    "sample_policy": "error_100_success_10"
  }
}

Manifest 字段 Schema

字段 要求 说明
schema 必填 Manifest schema 标识，例如 zeta.capability.manifest.v1。
name 必填 包、模块或 agent 稳定 ID，使用点分层级命名。
version 必填 Manifest 语义版本，能力边界收紧为 minor，放宽为 review-gated minor 或 major。
owner 必填 责任团队或组件名，用于审计和故障归属。
entry 必填 能力声明对应的入口文件或模块路径。
capabilities 必填 能力数组，每项必须包含 id、reason 和资源收窄字段。
tools 可选 本 manifest 暴露或依赖的 tool 名称白名单。
trace 必填 默认 trace 脱敏配置和采样策略，运行时配置只能进一步收紧。

授权配置格式

运行时授权配置描述“实际允许什么”。授权配置必须是 manifest 声明能力的子集，不能新增 manifest 未声明的 capability。

{
  "schema": "zeta.capability.authz.v1",
  "target": "docs.summarizer",
  "environment": "prod",
  "grants": [
    {
      "capability": "fs.read",
      "resources": {
        "paths": ["./docs/releases/**/*.md"]
      },
      "expires_at": "2026-12-31T23:59:59Z"
    },
    {
      "capability": "ai.chat",
      "resources": {
        "models": ["fast"],
        "max_tokens_per_call": 2048,
        "max_calls_per_minute": 30
      }
    }
  ],
  "deny": [
    {
      "capability": "process.spawn",
      "reason": "文档摘要不需要执行进程"
    }
  ]
}

权限匹配与收窄规则

  • 双重匹配：调用必须同时命中源码 uses、manifest capabilities 和运行时 grants。
  • 只允许收窄：运行时授权可以减少路径、域名、模型、token、次数和有效期，不能扩大 manifest 中的资源范围。
  • 显式 deny 优先：deny 命中的 capability 必须拒绝，即使其他 grant 允许。
  • 资源规范化：路径匹配前必须解析 .、..、符号链接和大小写规则；网络匹配前必须规范化域名、端口和协议。
  • 最长前缀优先：多个 grant 同时命中时使用最具体资源规则，并选择更低的预算、超时和 token 上限。
  • 用户授权再收窄：用户选择的文件、账户或会话范围是最终边界；部署授权不能覆盖用户撤销。

通配符禁用

Stable manifest 和生产授权禁止无边界通配符。开发环境可以使用临时宽授权，但必须带过期时间、审计标记和非生产环境约束。

写法 结论 替代方案
"*" 禁止 列出具体 capability，例如 fs.read、ai.chat。
fs.* 禁止 拆成 fs.read、fs.write，并分别声明路径。
./** 禁止用于生产 限制到业务目录，例如 ./docs/releases/**/*.md。
*.example.com 默认禁止 列出具体主机；确需子域时必须绑定协议、端口和组织级审批。
models: ["*"] 禁止 列出允许模型族和最大 token、费用、调用次数。

Schema 契约字段

字段 要求 说明
name 必填 稳定名称，使用 PascalCase。
version 必填 语义版本。破坏性变更必须提升 major。
fields 必填 字段名、类型、可选性、默认值和验证规则。
examples 必填 至少一个成功样例和一个失败样例。
compatibility 必填 向前兼容规则、废弃字段和迁移策略。

Tool 契约字段

字段 要求 说明
input 必填 输入 schema，禁止未结构化 Any 作为正式接口。
output 必填 输出 schema，AI 输出必须通过校验。
errors 必填 错误枚举，必须区分权限、超时、校验和外部服务失败。
uses 必填 最小 capability 集合。
timeout 必填 默认超时和最大超时。
idempotency 必填 标记为 none、read、retry_safe 或 key_required。
trace 必填 调用链字段、脱敏策略和采样策略。

Agent 契约字段

字段 要求 说明
tools 必填 Agent 可调用 tool 白名单。
memory 必填 状态存储类型、生命周期和隐私策略。
policy 必填 决策边界、人类确认点和禁止行为。
budget 必填 最大调用次数、token、费用、时间和重试次数。
explainability 必填 需要记录的计划、选择理由和最终决策摘要。

示例

schema Task {
  title: String
  priority: "low" | "medium" | "high"
}

tool classify(input: String) -> Result<Task, AIError>
  uses [ai.chat]
{
  ai.extract<Task>(model: "fast", input: input)
}

Agent 示例

agent SupportAgent {
  memory: SessionMemory(ttl: "24h")
  tools: [searchDocs, createTicket]
  policy: [
    "no_payment_action_without_user_confirm",
    "no_secret_exfiltration",
    "human_review_required_for_account_closure"
  ]
  budget: {
    max_tool_calls: 12,
    max_duration_ms: 30000
  }

  async fn handle(msg: Message) -> Result<Response, AgentError>
    uses [tool.call, ai.chat]
  {
    plan(msg).execute()
  }
}

错误模型

错误码 含义 是否可恢复
CAP_PERMISSION_DENIED 未声明或未授权 capability。 可通过配置或用户授权恢复。
CAP_RESOURCE_OUT_OF_SCOPE 请求资源超出 manifest、授权配置或用户授权边界。 需要缩小请求范围或申请更具体授权。
CAP_WILDCARD_REJECTED manifest 或授权配置包含禁止的通配符。 需要改为显式 capability 和资源列表。
CAP_SCHEMA_INVALID 输入或输出不符合 schema。 通常可通过重试、修复输入或降级恢复。
CAP_TIMEOUT 超过 tool 或 agent 的时间限制。 可重试，但必须受预算限制。
CAP_PROVIDER_UNAVAILABLE 外部模型、网络或工具服务不可用。 可降级到备用 provider 或排队。
CAP_POLICY_REJECTED 调用违反 agent policy 或部署策略。 需要用户确认、策略变更或终止任务。
CAP_BUDGET_EXCEEDED 超过费用、token、调用次数或时间预算。 需要新预算或人工确认。
CAP_TRACE_REDACTION_FAILED trace 脱敏失败或检测到敏感字段未被处理。 默认不可继续写入 trace；调用结果可返回但必须触发安全告警。

Trace Schema

Trace 事件记录 capability 调用的可审计事实。Trace 不是原始 payload 存储，禁止默认保存完整输入、完整输出、secret、凭证和个人敏感信息。

字段 要求 说明
trace_id 必填 全局调用链 ID，同一用户任务内保持稳定。
span_id 必填 单次 capability 调用 ID。
parent_span_id 可选 上游 tool、agent 或函数调用 ID。
timestamp 必填 UTC ISO-8601 时间戳。
caller 必填 调用方模块、函数、tool 或 agent 名称。
capability 必填 实际调用的 capability，例如 ai.chat。
resource 必填 规范化后的资源摘要，例如路径 hash、域名、模型名。
authz_decision 必填 allow、deny 或 narrowed，并记录命中的 grant ID。
input_summary 必填 输入摘要，包含 schema 名称、字段计数、字节数、hash 和脱敏后的短摘要。
output_summary 成功必填 输出摘要，包含 schema 名称、字段计数、字节数、hash 和质量校验结果。
error 失败必填 错误码、可恢复性、provider 错误分类和脱敏后的 message。
duration_ms 必填 调用耗时，不包含排队前的用户等待时间。
budget 必填 本次消耗的 token、费用、调用次数和剩余额度。
redaction 必填 脱敏 profile、命中规则数量、是否有降级或失败。
sampled 必填 是否落盘、采样原因和采样策略版本。

输入输出摘要

  • 结构摘要：记录 schema 名称、版本、字段数量、数组长度上限和 MIME 类型，不记录完整正文。
  • 内容摘要：默认记录前 160 个脱敏字符、字节数和稳定 hash；hash 使用部署级 salt，禁止跨环境关联用户数据。
  • 资源摘要：文件路径默认记录仓库相对路径或 hash，用户本地绝对路径必须脱敏用户名和临时目录。
  • 模型摘要：记录模型别名、provider、token 计数和参数范围，禁止记录完整 prompt 或 secret-bearing system message。
  • 失败摘要：记录错误码、错误分类和可恢复性；provider 原始错误只允许进入安全日志，不进入普通 trace。

脱敏策略

类别 处理规则 示例
Secret 完全替换为 [REDACTED_SECRET]，不保留长度。 API key、cookie、OAuth token、SSH key。
个人身份信息 按类型替换并保留最小调试信息。 邮箱替换为 [REDACTED_EMAIL]，手机号替换为 [REDACTED_PHONE]。
本地路径 保留项目相对路径；用户目录、临时目录和下载目录替换。 /Users/alice/project/a.md 记录为 [USER_HOME]/project/a.md。
业务正文 默认只保留短摘要和 hash；高敏 schema 只保留 hash。 合同、工单正文、用户上传文件内容。
安全失败 脱敏失败时 trace 写入失败事件，并禁止保存疑似原文。 返回 CAP_TRACE_REDACTION_FAILED。

采样策略

  • 错误全采样：权限拒绝、schema 失败、provider 失败、预算耗尽和脱敏失败必须 100% 记录。
  • 成功按比例：生产成功调用默认 10% 采样；开发和测试环境可 100% 采样，但仍必须脱敏。
  • 高风险全采样：process.spawn、secret.read、fs.write 和跨域网络写操作必须 100% 记录。
  • 高频限流：同一 trace 内相同成功事件超过阈值后只记录聚合计数、耗时分位和错误尾样本。
  • 采样可追溯：每条事件必须记录 sample_policy、策略版本和采样原因。

版本策略

  • Schema、tool 和 agent 契约使用语义版本。
  • 新增可选字段是 minor 变更。
  • 删除字段、改变字段含义、收紧验证规则、改变错误语义是 major 变更。
  • 破坏性变更必须提供迁移说明、兼容窗口和回滚方案。

Lint 与 Review 规则

  • 禁止 tool 缺少 schema、错误模型、timeout 或 uses 声明。
  • 禁止 agent 调用未列入 tools 白名单的 tool。
  • 禁止 capability 使用通配符授权进入 stable 文档。
  • 禁止 AI 输出未经 schema 校验进入业务状态。
  • Review 必须检查成功样例、失败样例、权限最小化和 trace 脱敏。

质量要求

  • Tool 必须声明 schema、错误模型、权限和超时。
  • Agent 必须声明可调用 tool 和决策边界。
  • 权限默认最小化，禁止隐式文件、网络、系统命令和 AI 模型访问。
  • Review 必须包含成功样例和失败样例。
  • Done 必须附带测试结果、文档链接和 DevGame 验收证据。
