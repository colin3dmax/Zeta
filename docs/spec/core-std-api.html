Zeta Core / Std API 边界

状态：accepted

更新时间：2026-06-04

适用范围：Zeta Stage 0 标准 API 命名空间、resolver 校验、用户文档和 Playground 示例。

验收标准：用户能明确知道当前哪些标准 API 路径可导入，未知路径会被编译器拒绝，并能在线验证。

API Boundary

先冻结可验证边界，再扩展真实标准库能力。

Zeta 不把尚未实现的标准库伪装成可用功能。Stage 0/1 先承诺最小命名空间边界：std.core 和 std.io 可以被导入；std.core 当前暴露整数专用的 OptionInt 和 ResultInt 枚举、自举 lexer/parser 所需的字符串 byte 扫描函数，以及 typed array builder；std.io 当前暴露 ResultString、文件读取、路径和诊断格式化函数；目录检查时，module graph 中存在的本地模块也可以被导入；其他路径必须被 resolver 明确拒绝。

设计决策

core 保持小

std.core 是最小语言核心能力的稳定入口，用于承载无需平台权限的基础 API。当前 Stage 0 暴露 OptionInt、ResultInt、字符串 byte 扫描函数和 typed array builder，先服务 enum payload、match、lexer/parser 自举、token/span 列表和 MIR 运行闭环；泛型 Option<T> / Result<T, E> / Vec<T> 等到类型系统支持后再开放。

std.io 保持受控

std.io 代表需要平台能力的 IO 层。当前只开放自举前端需要的 file_read_to_string、path_join、path_basename 和 diagnostic_format；文件读取用 ResultString 表达成功/失败，wasm 目标会返回 ResultString.Err。

未知路径必须失败

import std.net; 这类输入会得到 RESOLVE_UNKNOWN_IMPORT。这比静默接受更利于用户形成正确预期。

当前规则

路径 状态 说明
std.core 可导入 标准核心 API 边界，当前提供 OptionInt、ResultInt、字符串 byte 扫描函数和 typed array builder。
std.io 可导入 标准 IO API 边界，当前提供 ResultString、文件读取、路径拼接/取名和诊断格式化。
本地 module graph 目录检查可导入 zeta check <directory> 和 zeta run <directory> 会读取目录内 .zeta 文件，允许导入其中声明的模块；被导入模块里的 export fn 可在当前文件直接调用。
std.* 其他路径或缺失本地模块 拒绝 报告 RESOLVE_UNKNOWN_IMPORT，等对应 API 设计通过后再开放。

std.io 数据类型

类型 定义 说明
ResultString Ok(String) / Err(String) Stage 0 文件读取结果，用于在没有泛型 Result<T, E> 前表达源码读取成功或错误消息。

std.io 函数

函数 签名 说明
file_read_to_string (String) -> ResultString 原生运行时读取文件内容；读取失败返回 Err(message)。wasm 目标不访问宿主文件系统，返回 Err。
path_join (String, String) -> String 按需要用斜杠拼接两个路径片段。
path_basename (String) -> String 返回路径最后的文件名片段。
diagnostic_format (String, Int, Int, String) -> String 生成稳定诊断字符串：CODE at line:column: message。

std.core 数据类型

类型 定义 说明
OptionInt Some(Int) / None Stage 0 整数可选值，用于验证标准 enum、payload binding 和 match 穷尽性。
ResultInt Ok(Int) / Err(String) Stage 0 整数结果值，用于验证错误分支建模；真实错误体系后续由 capability、IO 和平台契约共同定义。

std.core typed array builder

函数 签名 说明
int_array_empty / string_array_empty / bool_array_empty () -> IntArray/StringArray/BoolArray 创建空 typed array。
int_array_push / string_array_push / bool_array_push (Array, Element) -> Array 返回追加一个元素后的新数组；配合 let mut 和赋值可表达 builder 风格追加。

std.core 字符串函数

函数 签名 说明
string_len (String) -> Int 返回 UTF-8 byte 长度。
string_byte_at (String, Int) -> Int 按 byte 下标读取一个 byte，返回 0-255 的 Int。
string_byte_slice (String, Int, Int) -> String 按 byte 起点和 byte 长度截取字符串；切分 UTF-8 字符边界会报运行时错误。
ascii_is_digit / ascii_is_alpha / ascii_is_alnum / ascii_is_whitespace (Int) -> Bool 按 ASCII byte 分类，非 0-255 输入返回 false。

示例

module demo.core;
import std.core;
import std.io;

export fn main() -> Int {
  return 42;
}
import std.core;

fn main() -> Int {
  let value: OptionInt = OptionInt.Some(42);
  match value {
    OptionInt.Some(answer) -> { return answer; },
    OptionInt.None -> { return 0; },
  }
  return 0;
}
import std.core;

fn main() -> Int {
  let mut values: IntArray = int_array_empty();
  values = int_array_push(values, 2);
  values = int_array_push(values, 4);
  return values[0] + values[1];
}
import std.core;

fn main() -> Int {
  let text: String = "A9 zeta";
  if ascii_is_alpha(string_byte_at(text, 0)) && ascii_is_digit(string_byte_at(text, 1)) {
    return string_len(text);
  }
  return 0;
}
import std.io;

fn main() -> String {
  let path: String = path_join("src", "main.zeta");
  return diagnostic_format("LEX_BAD_CHAR", 3, 5, path_basename(path));
}
import std.net;

fn main() -> Int {
  return 42;
}
// RESOLVE_UNKNOWN_IMPORT: unknown import `std.net`

后续阶段

  • Stage 1：实现跨文件模块图之前，继续只接受已登记标准路径。
  • Stage 1：为 std.core 定义不可分配、无需权限的基础 API 候选。
  • Stage 2：为 std.io 补齐权限、错误类型和平台目标差异。
  • Stage 2：开放第三方模块和包管理时，再让非标准路径进入模块解析。
