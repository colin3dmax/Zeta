Zeta HTML 文档规范

状态：accepted

更新时间：2026-05-29

适用范围：Zeta 仓库内正式设计文档的 HTML 格式、元信息、链接和交付规则。

验收标准：新增正式文档必须包含 title、h1、状态、更新时间、适用范围和验收标准，并使用相对链接。

格式规则

  • 除 GitHub README.md 使用 Markdown 作为仓库首页外，其他项目文档全部使用 HTML。
  • 正式文档优先中文，后续通过同一结构支持多语言版本。
  • 每份文档必须包含 title、h1、状态、更新时间、适用范围和验收标准。
  • 代码示例使用 pre/code，示例应能进入后续 golden tests。
  • 文档之间使用相对链接，避免不可迁移的本地绝对路径。
  • 后续多语言版本使用相同目录结构，例如 zh-CN、en-US。

验证命令

python3 tools/check-docs.py

该命令检查 HTML 标签配平、正式文档元信息、HTML 相对链接和 README 本地链接。

本地预览

python3 tools/serve-docs.py

默认地址是 http://127.0.0.1:8765/docs/index.html。预览服务会监听文档、源码、示例和工具文件变化，并自动刷新浏览器。

状态标记

  • draft：草案，可大幅调整。
  • accepted：已接受，作为实现依据。
  • stable：已被测试和实现验证。
  • deprecated：仍可参考，但不再作为新实现依据。

DevGame 交付规则

  • 任务进入 in_progress 前必须说明预期改动、风险和验证方式。
  • 任务进入 review 前必须附带测试结果和文档链接。
  • 任务进入 done 前必须处理 review 评论并记录最终验收证据。
  • 跨模块任务必须先定 schema 和规范，再实现 tool、agent 或 runtime 行为。

Owner 角色

Owner 职责 交付证据
project-lead 拆分任务、裁决范围、处理 review/done 闸门和跨角色交接。 DevGame 最终验收记录、关键提交、发布或回归证据。
compiler-architect 语言前端、诊断、AST/HIR/MIR、编译器 corpus 和 bootstrap 路线。 golden tests、CLI 输出、源码和规格文档链接。
runtime-platform-architect target matrix、runtime 分层、panic、allocator、CI 和平台适配。 target 表、CI matrix、build/smoke 记录和风险说明。
ai-runtime-architect capability、schema、tool、agent、policy、budget 和 trace 契约。 成功/失败示例、权限边界、脱敏规则和审计字段。
tech-writer README、用户文档、HTML 规范、链接和术语一致性。 文档链接、状态/更新时间、检查命令输出。
qa-release 发布前验收、线上 smoke、回归矩阵和发布记录。 测试命令、截图、线上 URL、版本或 artifact hash。

任务流转

  1. pending：任务已创建但未开始，必须包含目标、交付物和验收标准。
  2. assigned：任务已指定 owner，开始前确认影响范围、风险和验证方式。
  3. in_progress：正在实施，跨文件或跨模块改动必须持续记录关键决策。
  4. review：实现或文档已完成，必须附带测试结果、文档链接、风险和人工验收步骤。
  5. done：review 已处理，必须记录最终验收证据、关键文件、提交、PR、发布链接或截图。
  6. blocked：无法继续时必须说明阻塞原因、需要谁决策以及恢复条件。
