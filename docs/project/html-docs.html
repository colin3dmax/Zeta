Zeta HTML 文档规范

状态：draft

更新时间：2026-05-28

适用范围：Zeta 仓库内正式设计文档的 HTML 格式、元信息、链接和交付规则。

验收标准：新增正式文档必须包含 title、h1、状态、更新时间、适用范围和验收标准，并使用相对链接。

格式规则

  • 除 GitHub README.md 使用 Markdown 作为仓库首页外，其他项目文档全部使用 HTML。
  • 正式文档优先中文，后续通过同一结构支持多语言版本。
  • 每份文档必须包含 title、h1、状态、更新时间、适用范围和验收标准。
  • 代码示例使用 pre/code，示例应能进入后续 golden tests。
  • 文档之间使用相对链接，避免不可迁移的本地绝对路径。
  • 后续多语言版本使用相同目录结构，例如 zh-CN、en-US。

验证命令

python3 tools/check-docs.py

该命令检查 HTML 标签配平、正式文档元信息、HTML 相对链接和 README 本地链接。

本地预览

python3 tools/serve-docs.py

默认地址是 http://127.0.0.1:8765/docs/index.html。预览服务会监听文档、源码、示例和工具文件变化，并自动刷新浏览器。

状态标记

  • draft：草案，可大幅调整。
  • accepted：已接受，作为实现依据。
  • stable：已被测试和实现验证。
  • deprecated：仍可参考，但不再作为新实现依据。

DevGame 交付规则

  • 任务进入 in_progress 前必须说明预期改动、风险和验证方式。
  • 任务进入 review 前必须附带测试结果和文档链接。
  • 任务进入 done 前必须处理 review 评论并记录最终验收证据。
  • 跨模块任务必须先定 schema 和规范，再实现 tool、agent 或 runtime 行为。
